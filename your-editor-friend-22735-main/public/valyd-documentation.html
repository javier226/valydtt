FaceVector Technology

What are FaceVectors?

FaceVectors are mathematical representations of facial features that enable identity verification without storing actual photographs or raw biometric data. They are:

  • One-way transformations: Cannot be reversed to reconstruct the original image
  • Distributed: Stored across multiple servers in the company infrastructure
  • Encrypted in transit: Protected by TLS 1.3 and additional application-layer encryption
  • Rotatable: Can be regenerated if compromised, unlike fingerprints

User Node Architecture

Each user operates a secure node (typically their device) where all private identifiable information is stored:

Data Storage Strategy

  • Encryption: AES-256-GCM encryption with user-controlled keys
  • Sharding: Data is split into multiple encrypted shards stored across secure partitions
  • Access Control: Multi-factor authentication required for local access
  • Backup: User-controlled encrypted backups with key recovery mechanisms

Threat Model & Mitigations

Compromise of Company Servers

Risk: Attacker gains access to Valyd infrastructure

Mitigation:

  • FaceVectors alone cannot identify individuals without additional context
  • No PII is stored on company servers
  • Distributed architecture means compromising one server reveals minimal information
  • Immediate detection and response protocols

Compromise of User Device

Risk: Attacker gains physical or remote access to user device

Mitigation:

  • Data is encrypted even at rest on the device
  • Multi-factor authentication required for decryption
  • Sharding makes partial data access useless
  • Remote wipe capabilities for lost/stolen devices
  • Biometric authentication required for sensitive operations

Man-in-the-Middle Attacks

Risk: Attacker intercepts communications between user and Valyd

Mitigation:

  • TLS 1.3 with certificate pinning
  • Additional application-layer encryption
  • Mutual authentication between client and server
  • Perfect forward secrecy ensures past sessions remain secure

Privacy Features

  • Selective Disclosure: Share only the information required for verification

  • Audit Logs: Complete transparency into who accessed your data and when
  • Right to Deletion: Permanently delete your data from all systems
  • Data Portability: Export your data in standard formats

Quick Start

Integrate Valyd into your application in minutes with our SDKs and APIs:

# Install Valyd SDK npm install @valyd/sdk # Python pip install valyd-sdk # Go go get github.com/valyd/sdk-go

Basic Integration Example

JavaScript/TypeScript

import { ValydClient } from '@valyd/sdk'; // Initialize client const valyd = new ValydClient({ apiKey: 'your-api-key', environment: 'production' }); // Request verification const verification = await valyd.requestVerification({ userId: 'user-123', verificationType: 'identity', redirectUrl: 'https://yourapp.com/verify-callback' }); // Check verification status const status = await valyd.getVerificationStatus(verification.id); if (status.verified) { // User successfully verified console.log('User verified!'); console.log('Verification level:', status.level); }

Python

from valyd import ValydClient # Initialize client client = ValydClient( api_key='your-api-key', environment='production' ) # Request verification verification = client.request_verification( user_id='user-123', verification_type='identity', redirect_url='https://yourapp.com/verify-callback' ) # Check verification status status = client.get_verification_status(verification['id']) if status['verified']: print('User verified!') print(f"Verification level: {status['level']}")

Verification Flow

  1. Initiate Verification: Your application requests verification via Valyd API
  2. User Consent: User is redirected to Valyd interface to provide consent
  3. Identity Verification: User completes biometric verification on their device
  4. Cryptographic Proof: Valyd generates zero-knowledge proof of verification
  5. Callback: Your application receives verification token
  6. Validation: Your application validates the token with Valyd API

Legal & Compliance

Government Access & Subpoenas

âš–ï¸ Lawful Access Procedures

Valyd is designed to comply with lawful government requests while protecting user privacy to the maximum extent possible.

When Valyd May Provide Access:

  • Valid court order or subpoena with proper jurisdiction
  • Search warrant issued by a judge
  • Emergency requests involving imminent danger (with subsequent court validation)

What Can Be Accessed:

With a valid legal order, the following may be accessible:

  • User Node Data: Encrypted PII stored on user devices can be decrypted with proper legal authority
  • FaceVectors: Distributed biometric templates (though these alone cannot identify individuals)
  • Verification History: Timestamps and metadata about verification events
  • Account Information: Non-sensitive account metadata

What Cannot Be Accessed (Even with Court Order):

  • User data that has been deleted (we don't maintain backups of deleted data)
  • Information we don't collect in the first place
  • Data beyond the scope of the legal order

Frequently Asked Questions

General Questions

What makes Valyd different from other identity providers?

Valyd is built on a fundamentally different architecture. Unlike traditional providers that store your data on their servers, Valyd stores your private information exclusively on your device, encrypted and sharded. FaceVectors are decentralized across company infrastructure, meaning there's no central honeypot of user data to attack.

Can Valyd see my personal information?

No. Your private identifiable information is encrypted on your device with keys only you control. Valyd never has access to your decrypted personal data. We can verify your identity using zero-knowledge proofs without ever seeing the underlying information.

What happens if I lose my device?

You can recover your account using our secure backup and recovery system. Your encrypted data can be restored to a new device using your recovery keys. We recommend storing recovery keys in a secure location like a password manager or physical safe.

Security Questions

What is a FaceVector and can it be reversed?

A FaceVector is a mathematical representation of facial features used for verification. It's a one-way transformation - you cannot reconstruct a photograph from a FaceVector. Think of it like a cryptographic hash of your face that can verify you're the same person without storing your actual image.

How secure is my data if Valyd's servers are hacked?

Even if our servers were completely compromised, attackers would only obtain FaceVectors, which alone cannot identify individuals without additional context. All your personal identifiable information remains encrypted on your device. The decentralized nature of our FaceVector storage means compromising one server reveals minimal information.

What encryption do you use?

We use industry-standard AES-256-GCM encryption for data at rest and TLS 1.3 for data in transit. We also employ additional application-layer encryption and use ECDH for key exchange and ECDSA for digital signatures.

Privacy Questions

Can law enforcement access my data?

Only with a valid court order or subpoena issued by a judge with proper jurisdiction. Even then, access is limited to the specific data requested in the legal order. We challenge overly broad requests and notify users when legally permitted. See our Transparency Reports for statistics on government requests.

Do you sell my data?

Absolutely not. We never sell, rent, or share your personal data with third parties for marketing purposes. Our business model is based on verification services, not data monetization. We only share information when you explicitly consent or when legally required.

Can I delete my account and data?

Yes. You can delete your account at any time. This permanently removes your PII from your device and FaceVectors from our servers. We don't maintain backups of deleted data. Some audit logs may be retained for compliance purposes but contain no personally identifiable information.

Technical Questions

What platforms does Valyd support?

Valyd supports iOS, Android, Web (all modern browsers), Windows, macOS, and Linux. We provide SDKs for JavaScript/TypeScript, Python, Go, Java, Ruby, and PHP.

How fast is verification?

Typical verification takes less than 2 seconds. The actual biometric matching happens in milliseconds; most time is spent on network communication and user interface interactions.

What are the system requirements for users?

For biometric verification, users need a device with a camera (front or rear). Minimum requirements: iOS 14+, Android 8+, or any modern web browser with camera access. Approximately 50MB of storage space is required for the user node.

Can Valyd work offline?

Certain operations like local authentication can work offline. However, verification results need to be communicated to relying parties, which requires internet connectivity. We cache verification tokens for limited offline use.

Integration Questions

How long does integration take?

Basic integration can be completed in under an hour using our SDKs. More complex integrations with custom workflows typically take 1-3 days. We provide comprehensive documentation and developer support.

Do you offer a sandbox environment?

Yes. We provide a full-featured sandbox environment for testing and development. Sandbox API keys are available for free with no rate limits.

What's your uptime SLA?

We guarantee 99.9% uptime for Pro and Enterprise plans. Our decentralized architecture ensures high availability even if some servers experience issues.
